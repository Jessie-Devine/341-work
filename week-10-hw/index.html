Responsive Web Design

What is it, anyway?

Responsive web design is a type of web design that responds to users' behaviors, particularly size of the device with which they access the internet.

All of that, but mainly this:

See how this page is sort of grid-shaped? Gridding out a web layout really helps RWD be effective. Content is placed within individual containers, which are then free to move around the screen when it is resized. The grid doesn't have to be perfectly grid-like, either. As you can see, parts of this grid are blocks, some parts are tall and skinny, and others are short and fat. What's important is each piece of the grid being able to jump around and resize independently when needed.

The great thing about this is that content within individual containers also gets pushed around so it stays legible. Try resizing. See how the text moves to fit inside the container? This is only possible with RWD grids.

Float

One reason RWD works is the float tag. Float lets objects sit next to each other. For example, these black boxes are floating left, while our violet friend over here is floating right.

Relative Sizes

Absolute sizes don't work very well for RWD. Sizes instead need to be rendered in percentages, as in, percentages of screen space. It won't work for an object to be 1500px if your screen is half that size.

Margins

It is also helpful for margins to be relative. These black boxes have 5% margins on the right and left and 3% margins on the bottom. For the same reason relative sizes are better for RWD, relative margins work better than absolute margins.

Design

Design is also important for RWD. Each object should stand on its own, and stand out from the background. Additionally, the design should always be user-friendly and make sense, no matter what size the screen.
